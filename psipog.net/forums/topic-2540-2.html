Well, for my program, I was having it first authenticate the username/password with PsiPog via a PHP script. That script would return a password for the IRC channel. The client would connect to the IRC server, and join the channel with the correct password. While inside the room, it would communicate using RSA encryption scheme, using a custom protocol that I had invented. The other clients connected would be able to decode the messages, and display them as normal in everyone else's clients.

The protocol looked a lot like AIM's protocol, OSCAR. You can probably find some documentation around online to give you an idea on what goes into an IM client. Or you can look up the Jabber protocol, which was designed for IM use.
